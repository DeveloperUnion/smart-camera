# SmartCamera 仕様書

スマホブラウザで動く、リアルタイム物体検知 + カゴ追加アプリ。

## ユーザーフロー

1. ページを開く → 「カメラ開始」ボタンをタップ
2. 背面カメラが起動、画面いっぱいにライブ映像
3. 検知された物体に **グレー点線のバウンディングボックス + 日本語ラベル** が常時表示される
4. ユーザーがボックスをタップ → ボックスが **青の実線** に変化、カゴに +1
5. 同じ物体を再タップすると個数がさらに +1
6. 画面隅に `🛒 N` の件数バッジ
7. 「停止」ボタンでカメラ停止 → カゴ画面表示
8. カゴ画面: `ボトル × 3` 形式の一覧、各行に X で削除可
9. リロードでカゴはクリア (永続化なし)

## 技術スタック

| 領域 | 採用 |
|---|---|
| フロント | React 19 + Vite + TypeScript |
| 推論ランタイム | onnxruntime-web (WebGPU 優先 / WASM SIMD フォールバック) |
| モデル | YOLOv10n (COCO 80 クラス, uint8 量子化, ~3MB, NMS-free 出力) |
| カメラ | `getUserMedia` (`facingMode: 'environment'`) |
| 描画 | `<video>` + `<canvas>` オーバーレイ (Canvas 2D) |
| ラベル日本語化 | COCO 80 → JP 静的マッピング表 |
| トラッキング | なし (タップごとに個数加算するシンプル方式) |
| 状態管理 | React state (Zustand など導入しない) |
| デプロイ | Vercel (静的 SPA) |

## 動作環境

- iPhone 12 以降 (iOS Safari)
- Snapdragon 8 Gen 1 以降相当の Android (Chrome)
- HTTPS 必須 (`getUserMedia` の制約)

## パラメータ初期値

| 項目 | 値 |
|---|---|
| 推論入力解像度 | 640×640 (YOLOv10n 標準) |
| 推論 fps | 10〜15 fps (実機で間引き調整) |
| 信頼度閾値 | 0.5 |
| NMS | YOLOv10 は NMS-free のため不要 |

## 描画スタイル

- 未選択ボックス: `strokeStyle = '#9CA3AF'`、`setLineDash([6, 4])`、線幅 2px
- 選択済ボックス: `strokeStyle = '#3B82F6'`、実線、線幅 3px
- ラベル: ボックス上端に半透明背景 + 白文字

## 非機能要件

- カメラ映像はサーバー送信なし (オンデバイス完結)
- 初回モデルダウンロード ~3MB を Cache-Storage でキャッシュ
- バックグラウンド化・タブ閉じでカメラ停止 (リーク防止)

## 既知の割り切り

- 物体トラッキングは行わないため、同じ物体を何度でも追加できる (削除は停止後に X で行う)
- COCO 80 クラス外の物体は検知できない (将来 Gemini ハイブリッド化を検討)
- 録画機能なし (リアルタイム検知のみ、映像は保存されない)
